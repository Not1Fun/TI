# 视觉识别方案规划（OpenMV4 Plus + April Tag）

## 一、识别目标总览

| 目标 | 识别方式 | 负责模块 |
|------|---------|---------|
| 货物 A/B/C | April Tag（TAG36H11） | OpenMV4 Plus |
| 红色区域边框 | LAB颜色阈值 + 色块检测 | OpenMV4 Plus |
| 黑色循迹线 | 红外传感器阵列 | 主控（STM32） |

---

## 二、April Tag 识别方案

### Tag ID 分配

```
货物 A → Tag ID = 0
货物 B → Tag ID = 1
货物 C → Tag ID = 2
```

### 推荐参数

- 族：`TAG36H11`（抗误识别能力最强，优先选用）
- 分辨率：`QVGA (320×240)`，帧率快，适合实时检测
- 摄像头高度：距货物顶面约 15~20cm
- Tag 尺寸：建议贴 3cm×3cm 以上，保证在该高度下稳定识别

### 识别流程

```
OpenMV 上电初始化
    ↓
循环抓图 → find_apriltags()
    ↓
检测到 Tag → 提取 ID + 中心坐标 (cx, cy)
    ↓
通过 UART 封帧发送给 STM32
    ↓
STM32 解析后控制机械臂/夹爪定位抓取
```

---

## 三、红色区域识别方案

用于判断小车是否到达仓库区/卸载区，辅助停车定位。

- 色彩空间：LAB（受光照影响小于 RGB）
- 检测逻辑：检测到面积超过阈值的红色色块，视为进入区域，发送停车信号

> ⚠️ 阈值必须在**比赛类似光照条件**下重新标定，不可直接沿用测试值

---

## 四、OpenMV ↔ STM32 通信协议

串口波特率建议：115200

| 字节位置 | 内容 | 说明 |
|---------|------|------|
| Byte 0 | `0xAA` | 帧头 |
| Byte 1 | type | `0x01`=货物检测，`0x02`=红框检测 |
| Byte 2 | id | 货物ID：0/1/2 对应 A/B/C |
| Byte 3 | cx_H | 中心x坐标高字节 |
| Byte 4 | cx_L | 中心x坐标低字节 |
| Byte 5 | cy_H | 中心y坐标高字节 |
| Byte 6 | cy_L | 中心y坐标低字节 |
| Byte 7 | `0xBB` | 帧尾 |

---

## 五、摄像头安装建议

- 位置：小车前端悬臂，朝正下方俯视
- 高度：15~20cm（兼顾视野范围与识别精度）
- QVGA 在 15cm 高度时，视野约覆盖 20cm×15cm，足够扫描仓库区内全部 3 个货物

---

## 六、开发阶段划分

1. **阶段一**：April Tag 静态识别验证（单货物，确认 ID 正确）
2. **阶段二**：多货物同帧识别测试（3 个 Tag 同时出现）
3. **阶段三**：红色阈值标定（在实际/类比赛场地光照下）
4. **阶段四**：UART 通信联调（OpenMV → STM32 收发验证）
5. **阶段五**：整体联调（循迹 + 停车 + 识别 + 夹取）

## 七、可能踩的坑

### 1. 光照问题（高危）

- **坑**：题目明确说明现场可能有局部照明或室外自然光，且**不允许提出光照调整要求**
- **表现**：红色阈值偏移导致误检/漏检；April Tag 过曝或欠曝导致识别失败
- **对策**：
  - OpenMV 开启自动白平衡（`sensor.set_auto_whitebal(True)`）和自动增益（`sensor.set_auto_gain(True)`）
  - 红色阈值用 LAB 色彩空间，并在不同光照条件下多次标定取并集
  - 货物上方可加遮光罩（小型纸板），减少环境光直射 Tag

### 2. April Tag 识别距离不足

- **坑**：摄像头安装过高，或 Tag 尺寸太小，导致识别率低
- **表现**：小车到仓库区后 Tag 时有时无，位置抖动大
- **对策**：
  - Tag 尺寸至少 3cm×3cm
  - 摄像头高度不超过 20cm
  - 用 `tag.decision_margin()` 过滤低置信度结果（建议阈值 > 15）

### 3. 多 Tag 同帧干扰

- **坑**：3 个货物同时在视野内，位置相近时 Tag 角点可能相互干扰
- **表现**：某个 Tag 的 ID 被错误识别为相邻 Tag
- **对策**：
  - 货物摆放间距保证 ≥ 3cm
  - 识别时以 `tag.id()` 为主，结合坐标排序确认对应位置（左/中/右）

### 4. UART 丢帧 / 乱序

- **坑**：OpenMV 连续发送，STM32 来不及处理，导致帧错位解析
- **表现**：STM32 收到的货物 ID 乱序，夹错货
- **对策**：
  - 加帧头帧尾校验（`0xAA ... 0xBB`）
  - STM32 用状态机解析，收到完整帧才处理
  - OpenMV 每次识别成功后加短延时（50ms），避免刷屏

### 5. 红色框识别误触发

- **坑**：场地白布上若有红色反光、阴影偏色，可能误触发"进入区域"信号
- **表现**：小车还没到仓库区就提前停车
- **对策**：
  - 增加面积阈值过滤（色块面积需 > 500 像素才触发）
  - 连续多帧（≥3帧）检测到才发送停车信号

### 6. 摄像头震动导致识别抖动

- **坑**：小车行进时震动，帧图模糊，Tag 无法识别
- **表现**：运动中识别不到 Tag，停下来才能识别
- **对策**：
  - 仅在小车**停稳后**（主控发送"已停车"信号）才开启 OpenMV 识别
  - 或硬件上给摄像头加减震海绵垫

### 7. 货物摆放位置不固定

- **坑**：题目要求"各组自行摆放货物，不得在地图做标记"，位置每次不同
- **表现**：如果依赖固定坐标抓取会失败
- **对策**：
  - 必须用视觉反馈定位，不能写死坐标
  - OpenMV 返回 Tag 中心偏移量，主控做闭环校正后再抓取

### 8. 两次任务总时间限制 300 秒

- **坑**：识别和通信耗时累积，可能导致超时
- **表现**：任务二因时间不够被扣分
- **对策**：
  - OpenMV 识别完成后立即发送，不做多余等待
  - 减少 `sensor.skip_frames()` 帧数（够用即可，不要设太大）
  - 主控收到识别结果后立即执行，不轮询等待

---

## 八、备选方案

若 April Tag 识别在比赛现场稳定性不足，可临时切换为：

- **颜色识别**：货物 A/B/C 分别贴红/绿/蓝色贴纸，用 LAB 色块检测
- **字符识别**：货物顶面写大号字母 A/B/C，用 `find_template()` 模板匹配

> 建议 April Tag 为主方案，颜色识别作为备用，两套代码均提前验证
